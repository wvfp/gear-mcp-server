## 核心组件

### 1. 入口层 (src/index.ts)

MCP 服务器的主入口，负责：
- 初始化 MCP SDK 服务器
- 注册工具、资源、提示处理器
- 管理 Godot 进程生命周期
- 参数名称映射 (snake_case ↔ camelCase)

### 2. 工具系统 (src/tools.ts + src/tool-definitions.ts)

**工具暴露配置**：

| 配置文件 | 核心工具 | 动态工具组 |
|---------|---------|-----------|
| `compact` (默认) | 33 个 | 22 组 (78 个按需激活) |
| `full/legacy` | 110+ | 无 (全部暴露) |

### 3. Godot Bridge (src/godot-bridge.ts)

内部 HTTP/WebSocket 服务器，与 Godot 编辑器/运行时通信：
- 端口：默认 6505
- 处理工具调用和结果回传
- 心跳保活机制

### 4. LSP 客户端 (src/lsp_client.ts)

连接到 Godot LSP 服务器：
- 端口：6005
- 提供代码诊断、自动完成、悬停信息

### 5. DAP 客户端 (src/dap_client.ts)

连接到 Godot 调试适配器协议：
- 端口：6006
- 支持断点设置、单步执行、变量检查

## 通信流程

### 场景编辑流程
```
MCP 客户端 → tools/add_node → 服务器 → Bridge (HTTP) → godot_mcp_editor 插件 → Godot 编辑器
                                    ↓
MCP 客户端 ← tools/result ← 服务器 ← Bridge (HTTP) ← Godot 编辑器 ←
```

### 调试流程
```
MCP 客户端 → dap_set_breakpoint → 服务器 → DAP 客户端 (TCP:6006) → Godot DAP
MCP 客户端 ← dap_stack_trace ← 服务器 ← DAP 客户端 ← Godot DAP
```

## 工具组机制

### 动态工具组

Gear 采用动态工具组机制来解决工具数量庞大但上下文有限的问题：

```
工具组激活流程：
1. 用户使用 tool.catalog 搜索
2. 匹配的工具组自动激活
3. MCP 客户端收到 notifications/tools/list_changed
4. 工具列表自动刷新
```

### 工具组列表

| 工具组 | 说明 |
|--------|------|
| `scene_advanced` | 节点复制、重新父化 |
| `runtime` | 运行时场景检查 |
| `dap` | 断点调试 |
| `lsp` | 代码补全、悬停 |
| `testing` | 截图、输入注入 |
| `animation` | 动画制作 |
| `resource` | 材质、着色器 |
| `navigation` | 导航系统 |
| ... | ... |

## 环境变量

| 变量 | 用途 | 默认值 |
|------|------|--------|
| `GEAR_TOOL_PROFILE` | 工具暴露模式 | `compact` |
| `GODOT_PATH` | Godot 可执行文件路径 | 自动检测 |
| `GODOT_BRIDGE_PORT` | Bridge 服务端口 | `6505` |
| `GEAR_TOOLS_PAGE_SIZE` | 工具列表分页大小 | `33` |

## 设计原则

1. **模块化**：各模块职责清晰，便于维护
2. **动态加载**：按需激活工具，控制 Token 消耗
3. **多协议集成**：Bridge (HTTP/WS) + LSP + DAP 覆盖所有场景
4. **向后兼容**：compact/full/legacy 多模式支持平滑过渡
5. **容错设计**：重试机制、熔断器、结构化日志
